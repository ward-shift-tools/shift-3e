# ICU看護師シフト自動生成システム 仕様書 & 運用マニュアル

最終更新: 2026-04-17

---

## 1. システム概要

ICU看護師の月次シフト表を、Tier制に基づく制約下で自動生成するMILP（混合整数線形計画）ベースの最適化システム。

- **エンジン**: PuLP + HiGHS ソルバー
- **UI**: Streamlit（ブラウザ）/ CLI
- **入出力**: Excel（.xlsx） / Google スプレッドシート
- **デプロイ**: Streamlit Cloud（GitHub連携）

### 構成ファイル
| ファイル | 役割 |
|---|---|
| `shift_scheduler.py` | ソルバー本体・制約定義・Excel/GSheet入出力 |
| `app.py` | Streamlit UI（アップロード・プレビュー・結果表示） |
| `create_test_data.py` | テスト用データ生成スクリプト |

## 2. Tier制度（5段階）

看護師の経験・責任レベルによる階層分け。夜勤ペアリングやリーダー配置の基本単位。

| Tier | 定義 | 日勤リーダー単独可 | 夜勤リーダー可 | 夜勤ペア制約 |
|---|---|---|---|---|
| **A** | ベテラン・リーダー格 | ✅ | ✅ | 自由 |
| **AB** | 中堅・リーダー代行可 | ❌ | ✅ | A欠時にバックアップ（ペナルティあり） |
| **B** | 一人立ち済み | ❌ | ❌ | **B+B禁止、B+C族禁止** |
| **C+** | C既卒（経験1〜2年相当） | ❌ | ❌ | A/AB/B下で夜勤可、C族同日1人まで |
| **C** | 新人・経験浅い | ❌ | ❌ | 必ずA/AB/Bと夜勤ペア、C族同日1人まで |

### 2.1 夜勤ペアの許可／禁止マトリクス
（その日の夜勤2人の組合せ。training除く）

| | A | AB | B | C+ | C |
|---|---|---|---|---|---|
| **A** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **AB** | ✅ | ⚠️100 | ⚠️100 | ⚠️50 | ⚠️200 |
| **B** | ✅ | ⚠️100 | ❌ | ❌ | ❌ |
| **C+** | ✅ | ⚠️50 | ❌ | ❌(C族≤1) | ❌(C族≤1) |
| **C** | ✅ | ⚠️200 | ❌ | ❌(C族≤1) | ❌(C族≤1) |

- ✅ = 許可
- ⚠️N = ソフトペナルティ（重みN）
- ❌ = ハード禁止（解不可能）

## 3. シフト種別

### 3.1 勤務種別
| 記号 | 名称 | 区分 | 時間 | 備考 |
|---|---|---|---|---|
| **日** | 日勤 | 日勤系 | 8:45〜17:00 | 標準 |
| **夜** | 夜勤 | 夜勤系 | 16:30〜翌9:30 (16h) | 翌日「明」必須 |
| **明** | 明け | 中間 | — | 夜勤翌日、公休にはカウントしない |
| **休** | 公休 | 休 | — | 月N日確保（13日標準） |
| **研** | 研修 | 勤務 | — | 勤務日数にカウント、日勤最低人数には含まない |
| **委** | 委員会 | 日勤系 | — | 勤務中に離席、**当日A/AB同席必須（ハード）** |
| **暇** | 休暇 | 休 | — | 有給・忌引等、公休と別枠 |

### 3.2 拡張シフト（設定でON/OFF）
| 記号 | 名称 | 時間 | 用途 |
|---|---|---|---|
| 早 | 早出 | 7:00〜16:00 (8h) | 早出対応 |
| 遅 | 遅出 | 12:00〜21:00 (8h) | 遅出対応 |
| 短 | 時短 | 8:45〜16:00 (6.25h) | 育児・介護対応 |
| 長 | 長日勤 | 8:45〜21:00 (12h) | 翌日休推奨 |
| 準 | 短夜勤 | 17:00〜翌5:00 (12h) | 夜勤系・翌日明け |

### 3.3 勤務希望記号
| 記号 | 意味 | 制約タイプ |
|---|---|---|
| 日 | 日勤希望 | ソフト |
| 夜 | 夜勤希望 | ソフト |
| 休 | 休希望 | ソフト |
| 研 | 研修指定 | **ハード** |
| 委 | 委員会指定 | **ハード（I=1強制 + 当日A/AB同席必須）** |
| 休暇 | 有給等 | **ハード（V=1強制）** |
| 夜不 | 夜勤不可 | **ハード（N=0強制）** |
| 明休 | 明or休 | **ハード** |

※希望は **月7日まで** を推奨（超過時は先頭7日のみ採用）

## 4. 制約体系

### 4.1 ハード制約（絶対遵守）

#### スタッフ個人
- 1日1シフトのみ
- 夜勤→翌日は明必須
- 明の翌日は休（専従は夜・休OK）
- 休→翌日夜勤禁止（通常スタッフ）
- 前月繰越: 前月末が「夜」→ 月初日=明、「明」→ 月初日=休
- 連勤最大5日（設定可、超過禁止）
- 公休日数 ±0〜1日以内（例: 13〜14日）
- 夜勤Min以上、Max以下（Max=0は絶対夜勤禁止）
- 時短: D/N/SN/LD全禁止、ST のみ可
- パートタイム: N/SN/R禁止、週勤務日数に基づく月間目標（±1）

#### 日次
- **夜勤人数 = 2人/日**（研修時は3人 = 2人通常 + 1人研修）
- **日勤最低人数 ≥ 5（ハード下限=4）**
- **日勤最低人数（新人除く）≥ 4（ハード下限=3）**
- **日勤リーダー（Aのいずれか） ≥ 1 日勤系/日**
- **夜勤リーダー（A または AB、夜勤可能者） ≥ 1 夜勤系/日**
- **C族（C+∪C）同日夜勤 ≤ 1**
- **B 同日夜勤 ≤ 1**
- **B + C族 同日夜勤禁止**
- **新人夜勤 → その日のA夜勤必須**
- **研修夜勤 → その日のA夜勤必須**
- 休暇(V)は希望指定された日のみ可
- 委員会(I)は希望指定された日のみ可
- **委員会(I)のある日は A or AB が純粋日勤系（委以外）に1人以上必須**
  （委員会参加者は勤務中に離席するため、当日リーダー級のバックアップを確保）

#### 専従
- 夜勤専従者: 希望がない限り日勤(D)禁止
- 夜明連続≤2回（N,A,N,A まではOK、その後は必ず休）

### 4.2 ソフト制約（ペナルティ付き）
目的関数で最小化する重み:

| 優先度 | 項目 | 重み |
|---|---|---|
| P0 | A夜勤リーダー欠 | 300 |
| P0 | 日勤最低人数不足（全体） | 200 |
| P0 | 日勤最低人数不足（新人除く） | 200 |
| P1 | 希望未達 | 250 |
| P2 | AB + 通常C ペア | 200 |
| P3 | AB + B ペア | 100 |
| P3 | AB + AB ペア | 100 |
| P4 | AB + C+ ペア | 50 |
| 均等化 | 夜勤回数の最大-最小差 | 重み変動 |
| その他 | 専従の夜勤推奨値乖離、連勤推奨超過 | 重み変動 |

### 4.3 設定可能な制約モード
| 項目 | モード | 意味 |
|---|---|---|
| 72時間規制 | strict / soft / none | 月間夜勤時間 ≤ 72h |
| 長日勤→翌日短夜勤 | strict / soft / none | LD の翌日SN強制 |
| 長日勤連続 | strict / soft / none | LD を連続させない |

## 7. 運用マニュアル

### 7.1 基本フロー
```
1. テンプレートDL  →  2. Excel/GSheetに記入  →  3. アップロード
     ↓                                                ↓
                                  4. 読み込みサマリーで意図通りか確認
                                                      ↓
                      5. 設定/希望タブで微調整  →  6. 生成ボタン
                                                      ↓
                                    7. 結果タブで検証（❌/警告がないか）
                                                      ↓
                                              8. Excel出力
```

### 7.2 アップロード後の確認ポイント
**「📋 読み込み結果サマリー」** パネルで以下を必ず確認:
- 🏷️ Tier構成が意図通りか
- 🚫 夜勤しない人（Max=0）に**新人・パート全員が含まれているか**
- 🎓 新人卒業日が正しいか
- ⏰ パートの週勤務日数
- 🌙 夜勤専従者
- 🏃 夜勤研修者の月回数
- 🎯 前月繰越の夜/明

論理エラー（❌）は生成前に必ず直すこと。

### 7.3 結果の見方
- **✓ 全日Aリーダー配置**: A夜勤欠けなし（理想状態）
- **✓ 日勤最低N人達成**: 不足日なし
- **✓ 全希望達成**: 希望未達なし
- **⚠️ 希望未達**: 優先度トレードオフの結果（許容範囲なら受け入れ）
- **❌ Infeasible**: 制約が強すぎ → 後述のトラブルシューティング参照

## 8. 注意点・落とし穴

### 8.1 入力の落とし穴
| 症状 | 原因 | 対策 |
|---|---|---|
| 夜勤Max=0 なのに夜勤が入る | セルが空欄→「設定値5」が適用される | 明示的に `0` を入力 |
| 新人4人なのに「夜勤しない人」に出ない | 夜勤Max 未入力 | Max=0 を明示入力 |
| Numbers→xlsx で 0 が空欄化 | Numbersの既知挙動 | 保存後、数値列は数値であることを確認 |
| 時短=◯ なのに週勤務未指定 | 通常は時短＝パート併用 | 時短なら週勤務も入力 |
| 希望「休暇」と曜日制限の衝突 | V=1 と O=1 の二重強制 | 修正済み（自動回避） |

### 8.2 制約の衝突しやすいパターン
| パターン | リスク | 回避策 |
|---|---|---|
| A夜勤可能者が3人以下に減る月 | 日勤リーダー＋夜勤リーダー両立困難 | 応援シフト・AB活用 |
| 同日に複数A/ABが希望休 | リーダー配置不能 | 希望をずらす |
| 新人が半数近い構成 | 夜勤可能者が枯渇 | 新人卒業日を計画的に設定 |
| 川本のような`夜勤Min=Max=4`固定 | 研修日程の自由度低下 | 研修日付は他の希望を避ける |
| 10日連続休暇 + 曜日制限あり | Infeasible（修正済みだが注意） | 休暇は曜日制限のない日に設定 |
| 同日に委員会3人以上 + A/AB大半が休 | A/AB同席バックアップ不能 | 委員会日は他のA/ABが勤務できるよう調整 |

### 8.3 運用上の注意
- **前月末の引継ぎ情報**（夜/明）は月初繰越ロジックで必須。忘れずに入力
- **祝日設定**は `jpholiday` で自動取得されるが、**振替休日や特別休日は手動追加**
- **勤務希望は月7日まで**。8日以上入れると先頭7日のみ採用される
- **◯ と ○**: 両方Trueとして判定されるが、統一推奨
- **複数人が同日に夜/日希望**: 全員が実現できない場合、ソフト制約として調整される

## 9. トラブルシューティング

### 9.1 Infeasible（解なし）のチェックリスト
発生頻度順:

1. **休暇と曜日制限の衝突**（V=1とO=1の二重強制） → 休暇日を働ける曜日に寄せる
2. **A/AB夜勤可能者の不足**（全員が特定日にオフ） → 希望の分散調整
3. **夜勤Max=0の人が多すぎる**（12人以下で62枠）
4. **公休日数が多すぎる**（勤務必要日数 > 可能日数）
5. **連勤Max違反**（前月末からの連続勤務）
6. **研修夜勤の日付指定** vs 他の制約

### 9.2 ソルバーログの読み方
```
✓ リーダー夜勤枠(40) >= 31日           ← 夜勤リーダーキャパ OK
日勤最低(新人除く): 4人/日（対象15人）  ← 対象人数チェック
夜勤研修: 1名 → 最初のN回=3人夜勤     ← 研修構成
夜勤配分: 枠62 - 専従10 = 通常52回    ← 専従以外で配分
  ÷ 15人 ≒ 3.5回/人                   ← 1人あたり平均
⚠ 南: 勤務可能10日 < 週3目標13日       ← パート目標調整
```

### 9.3 よくある警告
- `Tier '' 不正 → スキップ`: 凡例行の読み取り（問題なし）
- `⚠ 希望9日 > 上限7日`: 先頭7日のみ採用（仕様通り）
- `⚠ 勤務可能N日 < 週目標M日 → N日に調整`: パート目標が曜日/休暇で削減された

## 10. 技術的背景（参考）

### 10.1 最適化モデル
- バイナリ変数: `x[staff, day, shift_type]` ∈ {0, 1}
- 変数削減: スタッフ属性に応じて不要な組合せを事前除外（通常40-50%削減）
- 目的関数: 重み付きソフト制約合計の最小化
- 複数パターン生成: ランダム化＋制約追加で多様性確保

### 10.2 主要ファイル／関数
| 場所 | 役割 |
|---|---|
| `shift_scheduler.py::build_and_solve()` | ソルバー本体（約800行） |
| `shift_scheduler.py::_parse_staff_list()` | スタッフCSV/行パース |
| `shift_scheduler.py::SETTINGS_DEF` | 全設定の既定値 |
| `app.py::_render_load_preview()` | アップロード後サマリーパネル |
| `app.py::_parse_uploaded_excel()` | Excel → 内部データ変換 |

### 10.3 既知の設計上の選択
- **明(A)は公休(O)にカウントしない**: 運用慣習に合わせ、月の「休」は13日分を公休＝O で確保
- **C族同日≤1はハード**: 経験不足ペア防止のため緩和不可
- **希望日数制限7日**: 入力量ディレクティブ過多を防ぐ。8日目以降は無視
